Linux

Un système d'exploitation est un ensemble de logiciels de base que nous permet d'utiliser un ordinateur. Il est nécessaire, par exemple, pour la communication entre les composants matériels (carte réseau, processeur, etc) et l'utilisation de periferiques (souris et clavier). Linux est un système d'exploitation, au même titre que Windows et MAC OS. Il a été créé en 1991 à partir de la fusion des outils GNU, développés par l'américan Richard Stallman, et le noyau Linux, développé par le finlandais Linus Torvalds.

Ce système possède de nombreux avantages: il est stable, multi-utilisateur, gratuit, ouvert et ilimité. Linux possède un grand nombre de distributions ("versions") adaptées au bonheur de tout un chacun. Les versions les plus stables sont les preferés des professionnels (Debian et Red Hat). Les distributions adressés aux novices sont Ubuntu, Fedora et Mandriva. Elles ont la particularité d'intégrer par défault une interface graphique utilisateur (GUI, du anglais graphic user interface) et la procédure d'installation est simplifié. Ces deux caracteristiques ont contribué à la popularisation des distributions Linux. Nous utilisons UBUNTU lors de nos cours et travaux.

Si vous aimez taper des commandes dans un terminal, en voici quelques unes à essayer !

cd : Naviguer dans l'arborescence système

cd /home/desktop/

mkdir : Créer un dossier.

mkdir /home/desktop/monNouveauDossier

touch/gedit/nano : Editer un fichier texte.

touch /home/desktop/monDossier/monFichier

echo : Remplacer le contenu d'un fichier texte

echo /home/desktop/monDossier/monFichier "Hello World"

echo >> : Rajouter du contenu à un fichier texte

echo >> /home/desktop/monDossier/monFichier "Ligne 2"

cat : Afficher le contenu d'un fichier texte

cat /home/monDossier/monFichier

mv : Déplacer ou renommer des fichiers ou répertoires.

mv /home/desktop/monDossier/monFichier /home/desktop/monNouveauDossier

cp : Copier Coller des fichiers ou répertoires.

cp /home/desktop/monNouveauDossier/monFichier /home/desktop/monDossier

ls : Afficher les dossiers/fichiers dans le répertoire actuel

ls /home/desktop/monNouveauDossier

ls -a : Afficher les dossier/fichiers cachés dans le répertoire actuel

ls -a /srv

sudo : S'octroyer les permissions super-utilisateur pour une commande (à placer au début de la commande)

sudo su : S'octroyer les permissions super-utilisateur

sudo mkdir srv/monSuperDossier

rm/rmdir: Supprimer un fichier/dossier

rm /home/desktop/monNouveauDossier/monFichier

rm -dir -rf: Supprimer un dossier qui n'est pas vide (ATTENTION)

rm -dir -rf /home/desktop/monNouveauDossier

man: Ouvrir le manuel d'utilisation du terminal

A présent, parlons de Git

Git est un moyen de versionner ses fichiers, il est très utilisé dans le domaine professionnel pour sa simplicité et sa practicité

Installer Git

sudo apt-get install git

Une fois installé, vous pouvez commencer à interagir avec lui.

Créer un dépot git dans le workspace (à spécifier)

git init {name}

Créer un dépot git dans srv (à spécifier)

git init --bare {name}

Vérifier l'état des fichiers

git status

Afficher les logs git

git log

Placer de nouveaux fichiers sous suivi de version

git add

Envoyer le fichier dans le dépôt git

git commit

Afficher les logs de l'auteur "Host"

git log --author=Host>

Passer sur la branche "Master"

git checkout master

Après un commit permet de nommer notre version en "v1"

git tag v1

Supprimer le tag "v1" mais conserve la version

git tag -d v1

Se connecter au dépôt git

git remote add origin /srv/passerelle.git

Afficher les connexions

git remote

Envoyer la version sur la branche "master"

git push origin master

Dupliquer un dépot .git

git clone /srv/passerelle.git
